# PROMPTS

## AI summary prompt

The personalized audit summary is generated using the Anthropic API (claude-sonnet-4). When the API is unavailable, the app falls back to a templated summary generated by `generateSummary()` in `auditEngine.js`.

### Production prompt (for when API key is available)

```
You are a no-nonsense financial advisor reviewing an AI tool spend audit for a startup.

The user's team:
- Size: {teamSize} people
- Primary use case: {useCase}
- Tools audited: {toolList}
- Total monthly spend: ${totalMonthlySpend}
- Total potential monthly savings: ${totalMonthlySaving}
- Tools flagged for overspend: {flaggedTools}
- Tools already optimised: {optimalTools}

Write a 80–100 word paragraph summarising this audit result. Rules:
- Be direct and specific. Use the actual numbers.
- Don't start with "Based on" or "Your audit shows".
- If savings > $500/mo, mention that consolidating through discounted credits (like Credex) could add further savings on top.
- If savings < $100/mo or zero, be honest — say the stack is well-optimised, don't manufacture urgency.
- No bullet points. One paragraph only. Plain English.
- Tone: smart CFO giving a two-minute verbal summary, not a marketing email.
```

### Why this prompt was written this way

**Specific word count (80–100):** The results page has a fixed card for the summary. Shorter is fine; longer breaks the layout and overwhelms the user before they read the per-tool breakdown.

**"Don't start with 'Based on'":** First drafts of the prompt produced output starting with "Based on your audit..." and "Your audit shows..." — formulaic openings that read as AI-generated filler. The explicit prohibition removed them.

**CFO voice instruction:** Without this, the model defaulted to enthusiastic marketing copy ("Great news! You could save..."). The CFO framing produces direct, credible language that a finance person would trust.

**Honest handling of low savings:** Early prompt versions had the model try to find *something* to recommend even when the stack was optimal. This eroded trust. The explicit instruction to be honest about well-optimised stacks produces more credible output and builds user trust for future audits.

### What I tried that didn't work

**Prompt v1 — too vague:**
```
Summarize this AI tool audit in 100 words.
Audit data: {json}
```
Result: Generic, unstructured output. Included raw JSON field names in the response. No tone control.

**Prompt v2 — structured output attempt:**
Tried asking for JSON `{ summary: "...", keyInsight: "..." }`. The model would occasionally include commentary outside the JSON block, breaking the parser. Reverted to plain text.

**Prompt v3 — too many rules:**
Added 12 specific rules. The model started following some rules and ignoring others in unpredictable ways. Simplified to 5 clear rules with the most impactful constraints.

### Fallback template

When the API is unavailable or returns an error, `generateSummary()` in `auditEngine.js` produces a deterministic paragraph using the audit findings. It covers the same key points (savings amount, flagged tools, optimal tools, Credex mention for high savers) without requiring an API call. The fallback is tested implicitly by all form submissions in the current build (no API key configured).
